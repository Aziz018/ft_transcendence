<h2>Contributing to ft_transcendence</h2>

![grisageeddine](https://github.com/user-attachments/assets/e0648776-f628-47b0-a6f8-1f7518bcf55c)

Thank you for your interest in contributing to the **ft_transcendence** project! We welcome all contributions, from bug reports to new features. This document outlines the guidelines for contributing to this repository.

## How to Contribute

We encourage you to contribute in various ways:

*   **💬 Join the Discussions**: Share your insights, provide feedback, or ask questions on our discussion board.
*   **🐛 Report Issues**: If you find a bug, a security vulnerability, or have a feature request, please open an issue on our [Issues page](https://github.com/Aziz018/ft_transcendence/issues).
*   **💡 Submit Pull Requests**: If you've developed a new feature, fixed a bug, or improved existing code, we encourage you to submit a pull request.

## Contributing Guidelines

To ensure a smooth collaboration process, please follow these steps when submitting code contributions:

1.  **Fork the Repository**: Start by forking the project repository to your GitHub account.
2.  **Clone Locally**: Clone the forked repository to your local machine using a git client.
    ```sh
    git clone https://github.com/YOUR_USERNAME/ft_transcendence.git
    ```
    (Replace `YOUR_USERNAME` with your GitHub username)
3.  **Create a New Branch**: Always work on a new branch, giving it a descriptive name that reflects the nature of your changes (e.g., `feature/add-login`, `fix/broken-socket`).
    ```sh
    git checkout -b new-feature-x
    ```
4.  **Make Your Changes**: Develop and test your changes locally. Ensure your code adheres to the project's coding style and conventions.
5.  **Commit Your Changes**: Commit your changes with a clear and concise message. A good commit message describes *what* you did and *why* you did it.
    ```sh
    git commit -m 'Implemented new feature x.'
    ```
6.  **Push to Your Fork**: Push the changes from your local branch to your forked repository on GitHub.
    ```sh
    git push origin new-feature-x
    ```
7.  **Submit a Pull Request**: Go to the original project repository on GitHub and create a new Pull Request.
    *   Clearly describe the changes you've made and their motivation.
    *   Reference any related issues (e.g., `Closes #123`).
    *   Ensure your PR targets the `main` branch unless instructed otherwise.
8.  **Review**: Your Pull Request will be reviewed by the project maintainers. Be prepared to address any feedback or requests for changes. Once your PR is reviewed and approved, it will be merged into the `main` branch. Congratulations on your contribution!

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project, you agree to abide by its terms.

## Reporting Bugs

If you encounter any bugs, please report them by opening a new issue. Include as much detail as possible:

*   A clear and concise description of the bug.
*   Steps to reproduce the behavior.
*   Expected behavior.
*   Screenshots or code snippets if applicable.
*   Your operating system and browser information.

## Feature Requests

We love to hear your ideas for new features! If you have an idea, please open a new issue and describe:

*   The feature you'd like to see.
*   Why you think it would be valuable to the project.
*   Any potential design considerations.
